🐾 OpenClaw Wiederherstellung — Schritt für Schritt

Erstellt von Bolla am 23.03.2026 | Für Chris

Was gesichert ist: Alle Erinnerungen, Scripts, Konfigurationen und Einstellungen liegen in einem privaten GitHub-Repository und können auf jedem neuen PC wiederhergestellt werden.

Was du auf dem neuen PC brauchst

  • Windows mit WSL (einfach in PowerShell: wsl eingeben)
  • Node.js (für OpenClaw)
  • OpenClaw installiert
  • Git installiert (in WSL)
  • Python 3 installiert (in WSL)

Schritt-für-Schritt Anleitung

Schritt 1 — OpenClaw installieren

Installiere OpenClaw auf dem neuen PC ganz normal wie beim ersten Mal. Starte die Einrichtung, aber brich ab bevor du einen neuen Agenten erstellst.

Schritt 2 — Workspace aus GitHub holen

Öffne PowerShell und starte WSL mit dem Befehl wsl. Dann eingeben:

cd /home/bolla/.openclaw

mv workspace workspace_alt  (alte leere Sicherung)

git clone https://github.com/openclaw-bolla/openclaw-bolla.git workspace

GitHub fragt nach Zugangsdaten: Username eingeben, als Passwort das GitHub PAT verwenden.

Schritt 3 — Anthropic Token einrichten

Den aktuellen Anthropic API Key (von Robin per Mail) über das OpenClaw Onboarding-Interface eingeben — genau wie beim ersten Mal.

Schritt 4 — Microsoft E-Mail Zugang neu einrichten

Der OAuth-Token für das Outlook-Konto (ernstmandel@outlook.de) läuft ab und muss neu erstellt werden:

Sage Bolla einfach: "Richte den E-Mail-Zugang neu ein" — sie führt dich durch den Device-Flow.

Schritt 5 — Token Watcher starten

Den automatischen Token-Updater aktivieren:

bash /home/bolla/.openclaw/workspace/scripts/start_token_watcher.sh

Dann den Task Scheduler einrichten (Bolla macht das automatisch beim ersten Chat).

Schritt 6 — Fertig!

Öffne den OpenClaw Webchat. Bolla liest ihre Erinnerungen aus dem Workspace und ist sofort wieder voll einsatzbereit — mit allen Einstellungen, Erinnerungen und deinem Kontext.

⚠️ Wichtig: Das GitHub-Repository ist privat. Den GitHub Personal Access Token (ghp_...) sicher aufbewahren — z.B. im Windows-Passwortmanager. Ohne ihn kein Zugriff auf das Repo.
