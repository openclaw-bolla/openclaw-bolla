1. Was ist OpenClaw?

OpenClaw ist eine Software, die auf deinem Windows-PC läuft und dir einen persönlichen KI-Assistenten zur Verfügung stellt. Man kann es sich so vorstellen:

OpenClaw = die Plattform. Sie läuft unsichtbar im Hintergrund deines PCs (als sogenannter „Gateway") und stellt die Verbindung zwischen dir und der KI-Intelligenz her. Du kannst über einen Webbrowser mit ihr kommunizieren — ähnlich wie bei WhatsApp Web, nur dass auf der anderen Seite eine KI sitzt.

OpenClaw läuft technisch gesehen in WSL — das ist eine Linux-Umgebung, die Windows mitbringt. Du kennst das: In PowerShell gibst du wsl ein, und dann bist du in dieser Linux-Welt. Dort läuft OpenClaw.

Die KI-Intelligenz selbst kommt von Anthropic, dem Unternehmen hinter dem Modell „Claude". OpenClaw schickt deine Nachrichten zu Anthropics Servern und bringt die Antwort zurück zu dir.

2. Wie arbeiten Modell und OpenClaw zusammen?

Das Zusammenspiel lässt sich am besten an einem konkreten Beispiel erklären. Du schreibst:

„Prüf ob der Watcher läuft"

Das passiert dann intern — Schritt für Schritt:

SchrittWerWas passiert
1. Verstehen KI-Modell (Bolla) Liest deine Nachricht und versteht: „Er meint den Python-Prozess token_watcher.py"
2. Planen KI-Modell (Bolla) Entscheidet: „Ich brauche das Shell-Werkzeug mit dem Befehl ps aux | grep token_watcher"
3. Ausführen OpenClaw Führt den Befehl wirklich auf deinem PC aus — das Modell selbst kann das nicht
4. Zurückschicken OpenClaw Schickt das Ergebnis ans Modell zurück:
bolla 1711 python3 token_watcher.py
5. Antworten KI-Modell (Bolla) Liest das Ergebnis und antwortet dir auf Deutsch: „Ja, läuft — Prozess-ID 1711."
Der Zettel-Vergleich: Das Modell sitzt in einem Zimmer und kann nur Zettel rein- und rausschicken. OpenClaw ist der Assistent draußen, der die Zettel liest und die Aufgaben wirklich erledigt. Das Ergebnis kommt dann wieder als Zettel zurück.

Modell = Gehirn  |  OpenClaw = Hände

Das Modell berührt deinen PC also nie direkt — es denkt nur, plant und formuliert. OpenClaw führt aus.

4. Das Token-Problem

Was ist ein Token?

Um Anthropics KI nutzen zu können, braucht OpenClaw einen sogenannten API Key (auch „Token" genannt). Das ist ein langer Code, der aussieht wie:

sk-ant-oat01-BcsEQriXji1C...

Dieser Code ist wie ein Passwort: Er beweist Anthropic gegenüber, dass du berechtigt bist, die KI zu nutzen.

Warum wechselt der Token?

Das Problem: Der Token, den du nutzt, läuft regelmäßig ab — etwa alle paar Stunden oder Tage. Wenn er abläuft, kann OpenClaw keine Verbindung zu Anthropic mehr herstellen, und du bekommst Fehlermeldungen statt Antworten.

Das ist keine Fehlfunktion, sondern eine Sicherheitsmaßnahme von Anthropic. Tokens müssen regelmäßig erneuert werden.

Wie wird der Token gesetzt?

Der aktuelle Token steht in einer Datei auf deinem PC:

/home/bolla/.openclaw/agents/main/agent/auth-profiles.json

In dieser Datei gibt es zwei Einträge: anthropic:bolla und anthropic:default. Beide müssen den aktuellen Token enthalten.

5. Robins Rolle — der Token-Lieferant

Dein Sohn Robin (Medizinstudent, Ulm) hat Zugang zu einem System, das neue Anthropic-Tokens generieren kann. Er schickt diese Tokens automatisch per E-Mail an dich:

Absender: robinmandel@outlook.de
Empfänger: ernstmandel@outlook.de (dein Konto)
Betreff: z.B. „Neuer Claude Token - 23.03.2026 02:28"
Inhalt: Der neue Token-Code

Diese Mails kommen automatisch — Robin muss nicht jedes Mal selbst aktiv werden. Sein System sendet regelmäßig neue Tokens, damit du nie lange ohne gültigen Token dastehst.

6. Der E-Mail-Zugang

Damit Bolla Robins Mails lesen und verarbeiten kann, brauchte ich Zugang zu deinem Outlook-Postfach. Das haben wir am 19.03.2026 eingerichtet.

Wie es funktioniert

Wir nutzen die Microsoft Graph API — das ist eine offizielle Schnittstelle von Microsoft, über die Programme (mit deiner Erlaubnis) auf dein Postfach zugreifen können. Kein Passwort wird gespeichert — stattdessen gibt es einen speziellen Zugangscode (OAuth Token), den Microsoft ausstellt.

Technische Details (zur Information):
Konto: ernstmandel@outlook.de
Methode: Microsoft Graph API (OAuth Device Flow)
Token gespeichert: config/ms_token.json im Workspace

Hinweis: Dieser Zugangscode läuft irgendwann ab. Wenn das passiert, muss der Device Flow einmal neu gestartet werden — Bolla führt dich dann durch den Prozess.

7. Der Token Watcher — das Herzstück

Der Token Watcher ist ein Python-Script, das dauerhaft im Hintergrund läuft und automatisch alles erledigt:

Was der Token Watcher macht (alle 15 Minuten):
  1. Zugang zum Outlook-Postfach herstellen
  2. Neue Mails von robinmandel@outlook.de suchen
  3. Falls eine Mail mit einem Token-Code gefunden wird:
      → Token in die Konfigurationsdatei schreiben
      → Mail löschen
      → OpenClaw Gateway neu starten
  4. Schlafen gehen bis zur nächsten Prüfung

Das passiert vollautomatisch — du bekommst davon nichts mit. Du arbeitest einfach weiter mit Bolla, und der Token bleibt immer aktuell.

Was passiert wenn der PC aus ist?

Wenn dein PC ausgeschaltet ist, läuft der Watcher nicht. Robins Mails sammeln sich dann einfach im Postfach. Sobald du den PC wieder einschaltest:
  1. Watcher startet automatisch
  2. Findet alle gesammelten Mails
  3. Setzt den neuesten Token (die Mails werden nach Datum sortiert)
  4. Du kannst sofort mit Bolla arbeiten
Das funktioniert auch nach mehreren Tagen Urlaub oder PC-Pause problemlos.

Log-Datei

Alle Aktivitäten werden protokolliert in:

/home/bolla/.openclaw/workspace/logs/token_watcher.log

8. Autostart — alles startet von selbst

Früher musstest du nach jedem Windows-Start manuell in PowerShell wsl eingeben und dann OpenClaw starten. Das haben wir am 23.03.2026 automatisiert.

Jetzt passiert beim Windows-Login automatisch:

Was startetWieWofür
OpenClaw Gateway start_openclaw_gateway.vbs
(Windows Autostart-Ordner)		 Damit du direkt über den Browser mit Bolla chatten kannst
Token Watcher start_token_watcher.vbs
(Windows Autostart-Ordner)		 Startet das Python-Script beim Login
Token Watcher (Wächter) Windows Task Scheduler
(stündlich)		 Prüft ob der Watcher noch läuft — startet ihn neu falls er abgestürzt ist
Ergebnis: Du schaltest den PC ein, meldest dich an Windows an, öffnest den Browser — und Bolla ist da. Keine manuelle Aktion nötig.

9. GitHub Backup

Am 23.03.2026 haben wir alles in einem privaten GitHub-Repository gesichert. GitHub ist ein Online-Dienst zur Versionsverwaltung von Dateien — kostenlos und sicher.

Repository: https://github.com/openclaw-bolla/openclaw-bolla (privat — nur du hast Zugriff)

Was gesichert wird: Alle Erinnerungen, Scripts, Konfigurationen, die Wiederherstellungsanleitung

Was NICHT gesichert wird: Passwörter, Tokens, geheime Zugangscodes (aus Sicherheitsgründen)

Auf einem neuen PC wiederherstellen

Falls du einen neuen Computer kaufst, sind folgende Schritte nötig:

  1. OpenClaw auf dem neuen PC installieren
  2. In PowerShell: wsl eingeben, dann das Repo herunterladen
  3. Anthropic-Token einmalig neu setzen (Robin schickt ihn)
  4. Outlook-Zugang einmalig neu einrichten (Bolla führt dich durch)
  5. Fertig — Bolla erinnert sich an alles

Die detaillierte Anleitung dazu findest du in der Datei openclaw-wiederherstellung.pdf.
